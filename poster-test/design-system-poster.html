CODE Smart Campus - Design System

A showcase of reusable Web Components built for the interactive project presentation.

Core & Utility

Displays Font Awesome icons with options for size, color, and basic animation.

Usage:

Attributes:

  • name: (String) Font Awesome class names (e.g., "fas fa-brain"). Required.
  • size: (String) CSS font-size value (e.g., "2rem", "24px"). Optional.
  • color: (String) CSS color value (e.g., "red", "#ff0000", "var(--some-color)"). Optional.
  • animation: (String) Currently supports "pulse". Optional.

Displays a small, capsule-shaped tag, often used for keywords.

Usage:

Attributes:

  • text: (String) The text content of the tag. Required.

Emphasis & Annotations

A dashed, rotated stamp effect for highlighting status (e.g., Approved, Complete).

Usage:

Attributes:

  • text: (String) The text to display inside the stamp. Required.

Simulates a sticky note, useful for brief comments or lists. Can display simple text or a title with list items.

Usage (Simple Text):

Usage (Title & List):

Attributes:

  • text: (String) Simple text content (use *either* text *or* title/items).
  • title: (String) Title for the note (used with `items`). Optional.
  • items: (JSON String Array) A JSON string representing an array of list items (e.g., `'["Item 1", "Item 2"]'`). Required if using `title`.
  • pin-color: (String) CSS color for the thumbtack icon. Optional (defaults to red-ish).

Displays text within a speech bubble shape with an optional directional tail.

Usage:

Attributes:

  • text: (String) The text content of the bubble. Required.
  • direction: (String) Tail direction: "top", "bottom", "left", "right". Optional (defaults to "bottom").

Renders a basic grid-based floor plan layout.

Usage:

Attributes:

  • columns: (Number) The number of columns in the grid layout. Optional (defaults to 3).
  • rooms: (JSON String Array) JSON string of room objects. Each object needs `label` (String) and optionally `span` (Number, default 1). Required.

Content Display

A simple representation of a file folder with a label and optional icon.

Usage:

Attributes:

  • label: (String) The text label for the folder. Required.
  • icon: (String) Font Awesome class names for the icon. Optional (defaults to "fas fa-file-alt").

Displays information about a persona, including name, description, and an icon.

Usage:

Attributes:

  • name: (String) The name of the persona. Required.
  • description: (String) A short description. Required.
  • icon: (String) Font Awesome class names for the persona's icon. Required.

Displays a single piece of feedback, typically with source, text, and an icon.

Usage:

Attributes:

  • source: (String) Who or what provided the feedback. Required.
  • text: (String) The feedback content. Required.
  • icon: (String) Font Awesome class names. Optional (defaults based on source: user/teacher/comment).

Showcases a key achievement or highlight, with category, text, and icon.

Usage:

Attributes:

  • category: (String) The category of the highlight. Required.
  • text: (String) The description of the highlight. Required.
  • icon: (String) Font Awesome class names. Required.

Specialized Visuals

Renders a simple linear flow diagram based on node data.

Usage:

Attributes:

  • nodes: (JSON String Array) JSON array of node objects. Each needs `label` (String) and `icon` (String - FA class). Required.

Composite

A complex component representing a single event on a timeline. It dynamically renders other components (like stamps, notes, diagrams) based on its `visuals` data.

Usage:

Attributes:

  • date: (String) Date of the event. Required.
  • title: (String) Title of the event. Required.
  • description: (String) Main description text. Optional.
  • key-elements: (JSON String Array) List of key points for the event. Optional.
  • visuals: (JSON String Array) Array of visual objects to render. Each object needs a `type` (String, e.g., "stamp", "sticky-note", "diagram") and other attributes specific to that type. Optional.
